# saw-arch-firm-app

**Project Report – startSaw**

**Web Application Development Course**

---

### **Project Purpose**

To create a web application for the architecture studio NBAArchitects to showcase their projects to the public and gain visibility. The application allows registered users to publish their projects. External users can view the blog and perform searches to find projects of interest.

---

### **User Types**

There are two types of users:

* External user
* Registered user

The external user can browse the website, view the blog, and search within the blog content.

The registered user, in addition to the external user’s actions, can view their profile, edit their profile, change their password, and create blog content.

---

### **Application Architecture**

The application consists of HTML pages, CSS stylesheets, and JavaScript for client-side content rendering. On the server side, PHP scripts handle database connections and generate HTML responses to send to the client.

The database contains:

* A table for registered users
* A table for blog content

**Database schema:**

* **Users table:**
  Attributes include user ID (primary key), first name, last name, email, and password (stored as an MD5 hash in binary form).

* **Blog table:**
  Attributes include article ID (primary key), keywords (used for search), blog content, and publication date.

The *blog_content* field is of type text.
A full-text index is implemented on *keywords* and *blog_content* to facilitate text search.

---

### **Application Components**

#### **Home Page**

HTML page with full-screen video, a dropdown main menu on the left, and a quick login dropdown menu on the right. After login, the page reloads showing the username instead of the menu. The username is retrieved via an AJAX call. The menu is dynamically updated using JavaScript.

#### **Login**

A PHP script receives form data, sanitizes it, verifies credentials in the database, and creates a session with user ID and email. The user is redirected to the homepage with a personalized menu.

#### **Registration**

A PHP script processes and sanitizes form data, verifies credentials, and generates an HTML success message page.

#### **Show Profile**

A PHP script accesses session variables, verifies credentials, and generates a page displaying user profile information.

#### **Update Profile**

HTML form (client-side) and PHP script (server-side) to update user information. After updating, the user is redirected to the profile page.

#### **Update Password**

Similar structure to profile update. Password is updated only if credentials are correct and the new password matches confirmation.

#### **Blog Creation**

Implemented using the JavaScript library TinyMCE for rich text editing. Keywords are handled dynamically with JavaScript. Images can be uploaded in base64 format. Data is processed by a PHP script and stored in the database.

#### **Blog Visualization**

Blog posts are displayed one at a time using pagination, generated by a PHP script using prepared SQL statements.

#### **Blog Search**

Search bar on client-side. PHP script retrieves keywords, sanitizes them, and queries the database using full-text search. Results are displayed dynamically.

#### **Log Out**

PHP script destroys session variables and redirects to the homepage.

### **Input Sanitization**

All input data is validated in PHP:

* `trim()` removes extra spaces
* `mysqli_real_escape_string()` prevents SQL errors

Example:

```php
$newemail = "";
if (isset($_POST['email']) == true) {
    $newemail = $_POST['email'];
    $newemail = trim($newemail);
    $newemail = mysqli_real_escape_string($con, $newemail);
} else {
    $newemail = $email;
}
```

### **Prepared Statements**

Used for executing queries multiple times with different values:

```php
$page = isset($_GET['page']) && is_numeric($_GET['page']) ? $_GET['page'] : 1;
$query = "SELECT * FROM blog LIMIT ?,?;";
$stmt = mysqli_prepare($con, $query);
$limit = 1;
$start = ($page - 1);
mysqli_stmt_bind_param($stmt, 'ii', $start, $limit);
mysqli_execute($stmt);
```
